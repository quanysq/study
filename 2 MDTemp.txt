1. nvm 安装 nodejs 运行报错：exit status 1: 'D:\Program' 乱码
	原因：nvm安装到了有空格的路径上（D:\Program Files），导致切换失败
	Link: https://blog.csdn.net/mrhaoxiaojun/article/details/122505142
2. 在vscode终端运行 npm run dev 出错：npm : 无法加载文件 D:\Learn\NodeJs\nvm\npm\npm.ps1。未对文件 D:\Learn\NodeJs\nvm\npm\npm.ps1 进行数字签名。无法在当前系统上运行该脚本。有关运行脚本和设置执行策略的详细信息，请参阅 https:/go.microsoft.com/fwlink/?LinkID=135170 中的 about_Execution_Policies。
	解决：直接删除 D:\Learn\NodeJs\nvm\npm\npm.ps1，或者把它改名
	Link：http://www.caotama.com/1424076.html
3. Win10环境变量Path的开头处增加Java变量后，编辑多行变成单行？
	把 C:\WINDOWS\system32; 加到最前
	https://blog.csdn.net/m0_67297673/article/details/123843510
4. 网上邻居看得见局域网机器，打开时提示找不到网络路径
	原因：该机器是双网卡，一个IP在局域网内，另一个IP不在局域网内
	解决：TrueNas 删除其中物理卡(re0)的IP配置
5. 打开网上邻居，提示：网络发现已关闭,网络计算机和设备不可见

	https://g.pconline.com.cn/x/1553/15536456.html
6. git报错: unable to access ‘https://github.com/…’: OpenSSL SSL_read: Connection was reset, errno 10054
	原因：一般是因为服务器的SSL证书没有经过第三方机构的签署，所以才报错
	解决：运行命令 git config --global http.sslVerify false
	参考：https://blog.csdn.net/bblood307/article/details/120307064
7. 使用 threeBSD 出错：Right-hand side of 'instanceof' is not an object
	原因：应该是threejs版本的问题
	解决：使用 three v110 没有问题
	参考：https://blog.csdn.net/qq_41604498/article/details/116939823
8. TrueNas 系统机器突然关机后重启提示：	This is a FreeNAS data disk and can not boot system. System halted. 进入 BIOS 后没有找到启动系统 SanDisk 'Cruzer' Fit
	原因：猜测是机房太热导致机器主板出现故障，BIOS 找不到启动系统
	解决：将 CMOS 电池取出来后擦一擦过几分钟后重新放回，问题解决
	扩展：CMOS 电池取出放回可以消除 BIOS 密码和重置 BIOS 
	参考：https://www.truenas.com/community/threads/this-is-a-freenas-data-disk-and-can-not-boot-system-system-halted-after-every-update.95940/
9. VB6 的程序拷到 WinXP 上不能使用，提示 ActiveX 部件不能创建
	原因：没有 VB6 运行时库
	解决：安装 VB6 后解决

74. 物联网 上位机和下位机有什么区别
	上位机和下位机是指在一个系统中，完成不同任务的两个不同的计算机设备。其中，上位机一般是指较为高端的计算机，它负责控制、管理和监视整个系统，而下位机则是指实际执行任务的设备，它们一般是一些嵌入式系统、传感器、执行器等。
	下位机一般与上位机相比，具有以下特点：
		1. 硬件资源较少，主要用于执行特定的任务。
		2. 通常不具有操作系统，或者使用的是嵌入式操作系统。
		3. 与上位机通信的方式一般是通过串口、CAN总线、以太网等通信协议。
		4. 一般不具有人机界面，无法直接与用户进行交互。
		5. 代码编写一般使用低级语言，如汇编语言、C语言等。
	
	而上位机则具有以下特点：
		1. 硬件资源较为丰富，可以完成更加复杂的计算任务。
		2. 通常使用通用操作系统，如Windows、Linux等。
		3. 与下位机通信的方式一般是通过串口、CAN总线、以太网等通信协议。
		4. 通常具有人机界面，可以与用户进行交互。
		5. 代码编写一般使用高级语言，如C++、Java等。

76. 物联网 RS485 是什么？
	RS485是一种通用的串行通信总线标准，采用差分信号方式实现数据传输，在数据传输的高速、长距离应用中具有优良的抗干扰性和可靠性，因此被广泛应用于工业自动化控制系统、智能楼宇、智能家居等领域的数据传输。
	RS485总线的工作特点包括：
		1. 传输距离远：RS485总线信号传输距离可达1200~1800米，比RS232串行通信更远，可满足较远距离的数据传输需求。
		2. 传输速率高：RS485总线的通讯速率可达到10Mbps，比RS232串行通信快得多，可适用于高速数据传输应用。
		3. 抗干扰性能好：RS485采用差分传输方式，信号具有较高的信噪比，能有效抵御外部干扰，并且可支持多主机、多从机工作。
		4. 扩展性强：RS485总线支持多达128个节点，支持半双工通信，可方便灵活地扩展和配置系统。
	总之，RS485总线是一种高可靠性、高扩展性、抗干扰性能好的串行通信总线标准，在工业自动化、电力电气、楼宇控制、智能家居等领域得到了广泛应用。

		```
		"applicationUrl": "http://localhost:5001"
		```

		在项目的 Program.cs 文件中，可以通过 UseUrls 方法来指定端口号。
		例如，将端口号修改为 5001：
		```
		public static IHostBuilder CreateHostBuilder(string[] args) =>
			Host.CreateDefaultBuilder(args)
				.ConfigureWebHostDefaults(webBuilder =>
				{
					webBuilder.UseUrls("http://localhost:5001");
					webBuilder.UseStartup<Startup>();
				});
		```
	3. 使用命令行参数指定端口号
		在启动项目时，可以使用命令行参数来指定端口号。
		例如，使用 dotnet run 命令启动项目，并将端口号指定为 5001：
		```
		dotnet run --urls=http://localhost:5001
		```

	使用格式如下：
	```
	mvn install:install-file -Dfile=<path-to-file> \
							 -DgroupId=<group-id> \
							 -DartifactId=<artifact-id> \
							 -Dversion=<version> \
	```

	- `-Dversion`：jar包的版本号
	- `-Dpackaging`：jar包的类型，一般为jar或war
	- `-DgeneratePom`：是否生成pom文件，一般为true

	4. 对象访问权限：指定对对象的读写访问权限。
	5. 对象状态：提供关于对象状态的信息，例如对象是否可用、是否被激活等。

99. 【上位机开发】设备波特率是什么意思？
	设备波特率是指设备在一秒钟内传输的位数。它是一个衡量通信速度的指标，通常用单位“波特”（Baud）来表示，也可以用每秒传输的位数（bps，即bits per second），两者是等价的概念。

	如果一个设备支持的最高波特率是9600 Baud，则该设备每秒可传输9600位。不同的设备通常会支持不同的波特率，通常波特率越高，传输速度越快。

	在串口通信中，设备波特率是串口通信的重要参数之一。我们需要保证串口通信的两端（如一个嵌入式设备和一台电脑之间）采用相同的波特率设置，否则会导致数据传输出错。同时，还需要保证串口通信的两端采用相同的数据格式、校验方式等参数设置，才能确保通信成功。

99. 【上位机开发】Modbus 是什么？
	Modbus 是一种通信协议，是一种用于工业自动化控制领域的通用通信协议。Modbus 最早于1979年由Modicon公司（美国施耐德电气旗下的品牌）开发，已成为自动化设备设计与开发领域中的一个标准。
	Modbus 协议采用了主从式通讯方式，其中主机（通常是一个计算机或控制器）会向从机（一个或多个外部设备）发送指令以读写数据。Modbus 可以使用不同的通信介质和传输方式，包括串口、以太网、无线等各种不同的接口。
	Modbus 最常用的三种变体是：Modbus RTU、Modbus ASCII 和 Modbus TCP。其中，Modbus RTU 和 Modbus ASCII 是串行通信协议，常用于串口通信；而 Modbus TCP 是基于 TCP/IP 协议的一种网络通信协议，常用于以太网或 Internet 网络上的通信。
	Modbus 协议简单、灵活、可靠，特别适用于小型、低成本的工业控制和监控环境。它广泛应用于自动化生产线、机器人技术、环保设备、自动化测试设备、能源管理等领域。

99. MySQL SQL语句的参数
	MySQL SQL语句的参数指的是在SQL语句中使用的变量或占位符，用于动态地传递值。以下是一些关于MySQL SQL语句参数的事情：
	1. 参数可以用于各种SQL语句，如SELECT、INSERT、UPDATE和DELETE。
	2. 参数可以使用问号（?）或命名占位符（:name）表示。
	3. 参数可以在SQL语句中多次使用，以便重复使用同一值。
	4. 参数可以通过绑定变量或预处理语句来设置。
	5. 绑定变量是在执行SQL语句之前将变量的值绑定到参数上。
	6. 预处理语句是一种事先准备好的SQL语句模板，其中包含参数占位符，可以通过设置参数来填充。

	以下是一些关于MySQL SQL语句参数的代码示例：
	- 使用问号作为占位符的示例：
		```
		SELECT * FROM customers WHERE customer_id = ?;
		```
	- 使用命名占位符的示例：
		```
		SELECT * FROM customers WHERE customer_name = :name;
		```
	- 使用绑定变量的示例：
		```
		SET @customer_id = 1;
		SELECT * FROM customers WHERE customer_id = @customer_id;
		```
	- 使用预处理语句的示例：
		```
		PREPARE stmt1 FROM 'SELECT * FROM customers WHERE customer_id = ?';
		SET @customer_id = 1;
		EXECUTE stmt1 USING @customer_id;
		```

99. SNMP MIB 文件格式说明
	SNMP（Simple Network Management Protocol）MIB（Management Information Base）文件是一种文本文件，它用来描述网络设备所支持的管理对象及其属性，以帮助管理人员对网络设备进行监控和管理。其格式如下：
	1. 文件头部分
		头部分主要包括MIB文件的版本信息和模板文件引用等元素，其中有些元素是必需的，例如SNMP版本号、MIB文件名称等。
	2. 符号定义部分
		符号定义部分定义了在MIB文件中使用的变量名和OID（Object Identification）。其中，OID是环境中唯一的标识符，OID由一组数字表示，代表特定的管理对象及其属性。
	3. 对象类型定义部分
		对象类型定义部分定义了管理对象的类型及其属性信息，包括如下几种对象类型：
		- INTEGER，整数类型；
		- STRING，字符串类型；
		- - BITS，位串类型；
		- OBJECT IDENTIFIER，OID类型；
		- NOTIFICATION-TYPE，通知类型；
		- SEQUENCE，序列类型；
	4. 组定义部分
		组定义部分定义了一组相关的管理对象，使其可以以组为单位进行处理，便于查找和管理。

	总体来说，MIB文件的格式比较规范和统一，主要用于描述SNMP中的管理对象，包括对象的类型和特性等。
99. 上位机开发是什么

99. AP 和 AC 是指什么？
	在无线网络中，AP和AC是两个常见的网络设备。

	AP（Access Point）是接入点的缩写，通常被称为无线接入点。它是一种无线设备，具备传输和接收网络信号的功能，用来连接终端设备到无线网络（WLAN）中。AP用于提供无线信号覆盖，扩展无线网络的范围，同时还可以支持无线安全加密等功能。

	SMIv2使得SNMP协议能够准确地描述被管理设备中的对象和属性，提供了SNMP的基础。是SNMP数据结构中非常重要的组成部分。

99. 会计中的征收率是什么意思？
	会计中的“征收率”是指国家征收企业所得税、个人所得税、增值税等各种税种时所采用的税率。税率是规定税收征收金额的百分比，按照税法规定，在一定时期内必须按规定计算应缴税款。税率通常以百分数形式表示，并由国家税务机关根据国家税收政策来制定和调整，公司必须依照规定交纳相应的税款。
	以企业所得税为例，中国企业所得税的征收率根据企业的盈利状况而不同。假设某公司年度利润额在100万元以下，该公司可适用15%的税率，而如果公司年度利润额超过100万元，则该公司剩余部分应纳税所得额按照25%计征企业所得税。此外，不同的税种、不同的所得、不同的税务政策和地区等，对税率也存在差异。
	税率的高低与企业的税负有直接关系。通常来说，较低的税率对企业来说是有利的，因为税负降低可为企业释放更多的经济活力，带来更多的增长机遇。因此，企业应加强参与税收政策的研究和了解，以更好地优化企业财务状况和开展经营活动。

99. 【上位机开发】串口开发中的RTS和DTR是什么？
	在串口开发中，RTS和DTR是两个控制信号，在串口通信时用于控制数据传输。它们都是为了控制数据流而存在的。

	RTS（Ready To Send）指示数据发送准备就绪，即告诉外设接收器，数据准备好了，可以发送了。RTS信号是由控制设备发出的。

	DTR（Data Terminal Ready）表示数据终端准备就绪，即告诉外设发送器，数据终端（即计算机）准备好了接收数据。DTR信号是由计算机发出的。

	在使用RS232串口开发时，当RS232C接口的DTR和RTS信号于IBM AT或兼容机的串行口相互连接时，就可用于控制信号的流传。两机之间的RS232C接口线有4 根控制线,分别为RTS（1号 ），DTR（2号 ），DCD（3号 ），CTS（4号 ）。一个计算机RS232C端口具有2根缺省的控制线DTR和RTS，但在电气上连接计算机端的两根发射线（TX和RX）后，还可由计算机软件设置DTR,RTS,DCD以及CTS四个控制线的高低电平状态。
99. QPS 是指什么？
	QPS是Queries Per Second的缩写，中文意思为每秒查询率。它是一个衡量计算机系统性能和吞吐量的指标，用于表示一个系统每秒可以执行多少个查询操作。在计算机系统的设计和调优中，QPS常常被用作测试系统的性能和瓶颈的衡量标准之一。通常情况下，QPS越高，说明系统的性能越好。

	QPS的具体值取决于系统的硬件设备、软件设计、数据处理方式、并发量等多个因素。在实际应用中，需要针对不同的业务需求和场景，优化设计和调整系统参数，以获得更好的QPS值。同时，也需要注意控制查询负载，防止系统运行超载和崩溃。
99. 系统启动时提示 Checking Media
	当系统启动时出现 "Checking Media" 的提示通常意味着系统正在检查启动介质。这可能是由于以下几个原因：
	1. 检测硬盘驱动器：系统可能正在检测并识别连接到计算机的硬盘驱动器。这是启动过程中的一部分，确保系统可以访问正确的启动介质。
	2. 检测光驱或USB驱动器：如果系统配置中包含光驱或USB驱动器，则系统可能会检查这些驱动器以确定是否存在引导介质（如安装光盘或启动USB设备）。如果您没有插入任何引导介质，您可以在启动时按下相应的按键（通常是 F12 或 F2），以进入系统设置并更改启动顺序或禁用不需要的启动设备。
	3. 硬件问题：尽管较少见，但也有可能是由于硬件问题导致系统出现此提示。这可能包括硬盘驱动器故障、数据线连接问题等。您可以尝试检查硬盘驱动器的连接，确保其连接牢固。如果出现硬件故障的迹象，可能需要更换硬盘驱动器或寻求专业的硬件维修。

	
	如果 "Checking Media" 的提示长时间停留在屏幕上或没有进一步的进展，您可以尝试以下操作：
	1. 重新启动计算机：有时，重新启动计算机可以解决临时的启动问题。
	2. 检查启动设备：确保没有插入任何不需要的启动介质（如光盘或USB设备），并且启动顺序正确配置。
	3. 检查硬件连接：检查硬盘驱动器的数据线是否连接正常，确保其连接牢固。
	4. 恢复或修复操作系统：如果问题仍然存在，可能需要尝试恢复或修复操作系统。您可以使用操作系统安装介质启动计算机，并尝试进行修复或恢复操作系统。
